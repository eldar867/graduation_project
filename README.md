## Учебная практика, задание УП.02

### Задание 1. Описание темы дипломного проекта

#### Тема: Разработка веб-приложения для автоматизированной анонимизации персональных данных в структурированных наборах.
### Актуальность:
В условиях ужесточения регуляторных требований (152-ФЗ, GDPR) и роста числа инцидентов с утечкой данных, обеспечение конфиденциальности информации становится критически важным для организаций и исследователей. Многие компании, научные коллективы и стартапы нуждаются в безопасном обмене датасетами для аналитики, тестирования ПО и обучения ML–моделей, однако ручная обработка полей с персональными данными трудоёмка, субъективна и подвержена человеческим ошибкам. Отсутствие стандартизированных инструментов приводит к рискам штрафов, репутационным потерям и снижению скорости разработки. Автоматизированное веб-решение позволит формализовать процесс анонимизации, снизить операционные издержки, обеспечить воспроизводимость результатов и гарантировать соответствие базовым требованиям информационной безопасности на этапе предобработки данных.
### Целевая аудитория:
1. Специалисты по информационной безопасности – для быстрой подготовки тестовой среды и демонстрационных наборов данных без нарушения политики конфиденциальности.
2. Аналитики данных и Data Scientists – для безопасного использования реальных данных в исследовательских и учебных задачах.
3. Разработчики и QA-инженеры – для генерации реалистичных, но обезличенных датасетов при тестировании систем.
4. Студенты и научные сотрудники – для работы с открытыми данными в курсовых, дипломах и публикациях без риска раскрытия персональных данных.
5. Представители малого и среднего бизнеса – для самостоятельной подготовки отчётности и передачи данных контрагентам в обезличенном виде.
### Предполагаемые функциональные возможности:
Приложение будет реализовано в виде веб–интерфейса с последовательным рабочим процессом:
1. Загрузка и валидация данных – поддержка форматов CSV, XLSX; автоматическая проверка структуры, кодировки и наличия пустых строк.
2. Автоматическое определение полей с персональными данными – анализ заголовков и содержимого столбцов (ФИО, email, телефон, адрес, паспортные данные и др.).
3. Выбор методов анонимизации – пользователь может применять к каждому столбцу один из методов: маскирование, псевдонимизация, обобщение, добавление статистического шума, полное удаление.
4. Предварительный просмотр – сравнение исходных и обработанных строк в табличном виде до применения изменений.
5. Экспорт и аудит – скачивание анонимизированного файла в исходном формате, генерация краткого отчёта о применённых операциях.
6. Клиентская и серверная логика – валидация ввода, обработка ошибок, индикация статуса загрузки/обработки, адаптивный интерфейс.
Используемые технологии (предварительно):
	Frontend: HTML5, CSS3, JavaScript (ES6+). Библиотеки: PapaParse (парсинг CSV), Chart.js (визуализация статистики обработки).
	Backend: Python, FastAPI, pandas (работа с данными), Faker.
	Хранение и логирование: локальная файловая система для временных сессий, PostgreSQL для хранения метаданных операций.
	Инструменты разработки: Git, Docker (контейнеризация), VS Code, Postman (тестирование API).
	Документирование: Markdown, OpenAPI (для описания эндпоинтов).
### Ожидаемый результат:
В ходе выполнения дипломного проекта будет разработан и протестирован рабочий веб–интерфейс, реализующий полный цикл анонимизации структурированных данных: от загрузки файла до экспорта обезличенного набора. Исходный код, инструкция по развёртыванию и пояснительная записка будут размещены в Git–репозитории. Проект послужит основой для дальнейшего развития: интеграции продвинутых методов приватности, автоматической проверки соответствия требованиям 152-ФЗ/GDPR, поддержки больших датасетов и корпоративного использования. Результаты могут быть использованы в учебных процессах, научных исследованиях и практической деятельности ИТ–подразделений.
